# tests

## Objetivo
Suite automatizada de testes unitarios/integracao/contrato da aplicacao.

## Padroes obrigatorios
- Todo bug corrigido deve ganhar teste de regressao.
- Novas features devem incluir testes positivos e negativos.
- Cobertura minima global deve permanecer >= 85%.

## Organizacao
- Agrupar testes por dominio/feature.
- Nomes de testes devem explicitar comportamento esperado.
- Fixtures devem minimizar estado compartilhado e flakiness.
