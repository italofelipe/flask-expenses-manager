# Feature Contract Pack — J15-2

## Feature
- Ferramenta `parcelado vs à vista` — contrato backend canônico

## Endpoints
- `POST /simulations/installment-vs-cash/calculate`
  - público
  - recalcula a comparação e retorna `tool_id`, `rule_version`, `input` e `result`
- `POST /simulations/installment-vs-cash/save`
  - autenticado
  - recalcula no servidor e persiste em `simulations`
- `POST /simulations/{simulation_id}/goal`
  - autenticado
  - exige entitlement `advanced_simulations`
  - cria uma meta a partir de uma simulação salva
- `POST /simulations/{simulation_id}/planned-expense`
  - autenticado
  - exige entitlement `advanced_simulations`
  - cria despesa planejada a partir de uma simulação salva

## Bridge para meta
- request exige `title` e `selected_option`
- `selected_option` aceita `cash` ou `installment`
- `target_amount` default é derivado no servidor a partir da opção selecionada

## Bridge para despesa planejada
- `cash` exige `due_date`
- `installment` exige `first_due_date`
- quando `upfront_fees > 0`, a bridge cria:
  - grupo parcelado
  - transação adicional para custos iniciais

## Persistência
- `tool_id`: `installment_vs_cash`
- `rule_version`: `2026.1`
- persistência no domínio `simulations`
- `result` persiste recomendação, comparativo, premissas, indicator snapshot e cronograma

## Entitlement
- save: sem entitlement premium
- bridge `goal` e `planned-expense`: `advanced_simulations`
